Impostare un piano di manutenzione in Enterprise Manager

SQL Server mette a disposizione uno strumento (i piani di manutenzione) che consentono di eseguire operazioni di manutenzione sui database SQL quali, ad esempio, il controllo integrità, l'aggiornamento delle statistiche sugli indici ed, eventualmente, i backup. Per i backup si consiglia comunque di effettuare processi di backup distinti, senza far uso dei piani di manutenzione. La procedura sottostante descrive come si consiglia di realizzare un piano di manutenzione.

  1. Entrare in SQL Server Enterprise Manager e nella cartella Gestione scegliere la voce di menu Piani di manutenzione database, poi Nuovo piano di manutenzione...:

  2. Nella finestra Creazione guidata piano di manutenzione database scegliere Avanti:

  3. Nella finestra Seleziona database selezionare l'opzione Tutti i database utente per coinvolgere nel controllo tutti i database utilizzati da Business:

  4. Sulla pagina Aggiorna informazioni di ottimizzazione dati selezionare l'opzione Riorganizza pagine di dati indice, poi premere su Cambia per impostare la frequenza del processo di ottimizzazione:

  5. Si consiglia di impostare il processo almeno una volta alla settimana, quando non c'è attività sull'archivio e quando non ci sono altri processi schedulati (backup o altro), quindi premere su OK e nella pagina Aggiorna informazioni di ottimizzazione dati premere su Avanti:

  6. Sulla pagina Controlla integrità database selezionare l'opzione Controlla integrità database, poi premere su Cambia per impostare la frequenza del processo di controllo:

  7. Anche in questo caso si consiglia di impostare il processo almeno una volta alla settimana, quando non c'è attività sull'archivio e quando non ci sono altri processi schedulati, quindi premere su OK e nella pagina Controlla integrità database premere su Avanti:

  8. Sulla pagina Specifica piano di backup del database deselezionare l'opzione Includi backup del database nel piano di manutenzione, poi premere su Avanti:

  9. Sulla pagina Specifica piano di backup del log delle transazioni deselezionare l'opzione Includi backup del log delle transazioni nel piano di manutenzione, poi premere su Avanti:

  10. Sulla pagina Posizione report selezionare il flag Scrivi report in un file di testo nella directory per conservare in un file di log l'esito del piano di manutenzione del database:

  11. Nella pagina Cronologia piano di manutenzione deselezionare tutte le voci come sotto indicato:

  12. Quindi si preme Fine sulla pagina di riepilogo:

  13. A questo punto viene salvato il piano di manutenzione con i due processi schedulati, visibili dalla cartella Gestione, Agente SQL Server, Processi .

  14. Tenere sotto controllo il file di log salvato al punto 10.
