Gestione flussi

Nome programma: BSTEGSFL.DLL
Moduli richiesti: Tesoreria

Questo programma accede alla tabella dei flussi apportando variazioni o integrazioni per:
- valutare eventuali soluzioni a problemi di carenze finanziarie, ad es. spostando date di pagamenti o anticipando date previste di incassi, ecc.
- sfruttare appieno le proprie disponibilità nel breve periodo.

Solitamente vi si accede dopo aver eseguito la Stampa dei flussi finanziari.

Seleziona flussi

In fase di apertura di un flusso sono richiesti i seguenti dati.

Codice conto É il codice conto da controllare; lasciando 0 sono estratti tutti.

Codice elaborazione É il codice di elaborazione relativo al flusso da controllare, corrisponde al relativo campo indicato nel programma Generazione / Estrazione flussi finanziari.

Filtro valuta Inserire il codice valuta, lasciando 0 sono estratti i movimenti in valuta di conto.

Tipo pagamento Selezionare tra le possibilità proposte il tipo di pagamento desiderato.

Origine dati Selezionare tra le sorgenti dati proposte quelle che s'intendono visualizzare o modificare.

Filtro date Selezionare quali date considerare tra quelle proposte.

Data prevista = data del movimento; ad es. data di pagamento di una ricevuta bancaria 31/01/2015.

Data programmata = data in cui si presume che sarà eseguita l'operazione, ad es. la ricevuta bancaria sarà pagata il 03/02/2015.

Tipo pagamento Selezionare tra le varie proposte il tipo di pagamento desiderato. É possibile più di una scelta.

Gestione flussi:

Codice conto/descrizione Visualizza il codice e la rispettiva descrizione del conto a cui si riferisce il movimento finanziario, entrata o uscita.

Tipo risorsa Visualizza il tipo di risorsa a cui fa riferimento il movimento.

Dare/Avere Visualizza il segno dell'operazione, dare = entrata, avere = uscita.

Tipo pagamento É il tipo di pagamento utilizzato.

Importo É l'importo del movimento, entrata o uscita.

Importo valuta / Codice valuta / Cambio Attualmente non gestiti.

Probabilità Visualizza se il movimento è CERTO o PROBABILE.

Codice causale/descrizione Visualizza la causale utilizzata per generare il movimento e la rispettiva descrizione.

Data movimento prevista É la data in cui avverrà il movimento.

Data movimento programmata É la data in cui è programmato il movimento. É utilizzata per attribuire al movimento una data diversa, se si sa a priori che la data prevista non sarà rispettata. Di default è impostata uguale alla data prevista, la gestione di questo campo è manuale.

Data valuta prevista É la data valuta in cui avverrà il movimento.

Data valuta programmata É la data valuta in cui è programmato il movimento. É utilizzata per attribuire al movimento una data valuta diversa, se si sa a priori che la data prevista non sarà rispettata. Di default è impostata uguale alla data prevista, la gestione di questo campo è manuale.

Progressivo collegato Gestito in automatico, solo per movimenti, derivanti da movimenti finanziari previsti, di conti operanti entrambi in stato patrimoniale, visualizza il movimento collegato a quello su cui si è posizionati, cioè il movimento del conto di contropartita.

Contropartita É la contropartita collegata al movimento su cui si è posizionati.

Codice elaborazione (scenario) Visualizza il codice indicato nel campo "Scenario" nel programma "Generazione / Estrazione flussi finanziari".

Descrizione É la descrizione del movimento, descrive l'origine del movimento e altre utili informazioni ad identificare l'entrata o uscita.

Progressivo É il numero progressivo del movimento all'interno della tabella dei flussi.

Commessa CA/Descrizione Visualizza il codice della commessa associata al movimento e la relativa descrizione.

Sub Commessa CA Impostabile solo dopo aver indicato la commessa di CA, è il codice della sottocommessa, se presente, associata al movimento.
