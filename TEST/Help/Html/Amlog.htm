Log

In Business NET esistono diversi tipi di Log, atti a tracciare errori, monitorare l'accesso, ... I principali tipi che possiamo idetificare sono:

  • log di setup e aggiornamento;
  • log di aggiornamento archivi;
  • log ultime query eseguite;
  • log errori runtime;
  • log di programma;
  • log di esecuzioni batch;
  • log di attività;
  • log di accesso.

Log di setup e aggiornamento

Il programma d'installazione crea un file di log di tutto il processo di installazione. Tale file è presente nella root del disco di avvio di Windows (ad esempio C:\) e, qualora il programma di installazione termina correttamente, nella cartella dei file di Business (ad esempio C:\Bus).
Il file possiede il seguente nome 'Setup Log anno-mese-giorno #contatore.txt' (ad esempio 'Setup Log 2012-10-26 #002.txt').

In fase di aggiornamento dell'installazione tramite il programma degli aggiornamenti le informazioni riguardo l'aggiornamento vengono memorizzate nel file Busagg.log nella directory dei file di Business (solitamente C:\Bus).

Log di aggiornamento archivi

L'operazione di aggiornamento archivi degli strumenti di amministrazione di Business genera il file BusAdminTools.log nella directory dei file di Business (solitamente C:\Bus).

E' possibile ottenere per questo log, un maggiore dettaglio impostando nel registro di configurazione la proprietà AdminToolsFullLog = -1 (come indicato nell'immagine sottostante).

Log di ultime query eseguite

Business è in grado di memorizzare su file di log le ultime query eseguite sull'archivio procedura e sul database azienda, per creare e visualizzare tali file basta posizionarsi sul titolo della finestra di un programma qualsiasi, premere tasto destro del mouse e selezionare la scelta 'Visualizza ultime query eseguire'. Questi vengono memorizzati e visualizzati su file con nome del tipo QueryAz_nomeutente.log (per archivi azienda) e QueryPrc_.log oppure QueryPrc_nomeutente.log (per l'archivio procedura), sempre all'interno della directory dei file.

Log errori runtime

Tutti gli errori di runtime che solitamente vengono mostrati a video vengono comunque loggat tramite accodamento anche sul file su BusNet_error.log nella directory dei file di Business (solitamente C:\Bus).

Log di programma

Molti programmi elaborativi creano un file di log con informazioni relative all'elaborazione che riesca oppure no (come ad esempio la Contabilizzazione Doc. di Magazzino, Calcolo ammortamenti, ...). Il nome del file è dato solitamente dal nome del programma più il nome operatore che ha eseguito l'elaborazione, per esempio BNCECALC_admin.log, BNVEFADI_admin.log, BN__GREG_admin.log, ...

Questi log vengono memorizzati principalmente nella directory file di Business NET (solitamente C:\Bus) oppure nella Asc, generalmente al lancio di ogni nuova elaborazione il vecchio file viene sovrascritto.

Log di esecuzioni batch

Sono identici ai log di programma a parte il fatto che generalmente al lancio di ogni nuova elaborazione il log viene accodato al file già esistente. Tali file di log vengono creati con la dicitura nome programma seguito dalla parola batch e l'operatore (ad esempio BNVEPNFA_BATCH_roby.log), inoltre ad ogni avvio batch vengono loggati su file di nome BusNetBatch più un numero (ad esempio BusNetBatch_3100.log) e informazioni rigurdo la stringa di comando di avvio e al caricamento dei vari componenti.

Log di attività

I log delle attività permettono di vedere l'elenco delle attività eseguite, sia per qulle relative a modifiche e cancellazioni (per esempio di documenti, anagrafiche, ...), sia quelle relative all'esecuzione di elaborazioni massive (per esempio la contabilizzazione delle fatture, la stampa libro giornale in definitivo, ...). A differenza dei log già visti questi vengono memorizzati sul database di Business NET, ma per attivare tale gestione occorre impostare da registro le opzioni globali OPZIONI/ScriviActLogD e OPZIONI/ScriviActLogE (impostandole a -1) oppure settando la proprietà ScriviActlog sulla cartella OPZIONI dei singoli programmi. Per altre informazioni in merito vedi l'argomento Visualizzazione Log attività.

Log di accesso

Business NET permette anche di tracciare l'attività di accesso al sistema gestionale da parte dei cosiddetti amministratori del sistema (visto i provvedimenti del Garante della Privacy, del 27 novembre 2008, in vigore dal 15/12/2009). Anche questi come i log di attivitò a differenza degli altri log già visti vengono memorizzati sul database di Business NET. L'attivazione avviena tramite una nuova apposita opzione di registro globale OPZIONI/ScriviActLogL = -1 (0 = default). Tale opzione fa si che il programma di menù registri:

  • ogni volta che un utente entra in business (logon);
  • ogni volta che un utente esce regolarmente da Business (da menù);
  • ogni volta che viene lanciato un programma child da menù.

Tali registrazioni saranno visibili/interrogabili dal programa Visualizzazione Log attività.

Note finali

Le indicazioni/suddivisioni riportate in questo capitolo non sono una regola standard in tutto Business NET dei log prodotti, bensì rappresentano l'impostazione che maggiormente si può ritrovare, ma possono esistere sempre casi che non rientrano e non rispettano le indicazioni/suddivisioni sopra indicate.
