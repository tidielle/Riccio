Gestione dei backup da Enterprise Manager

E' possibile utilizzare Enterprise Manager per schedulare i backup dei database SQL Server 2000.

La seguente procedura può essere adottata per impostare i backup dei database utilizzati da Business.

  1. Entrare nell'Enterprise Manager e per il database oggetto di backup selezionare la voce di menu Tutte le attività, Backup database...:

  2. Nella pagina Generale impostare il tipo di backup Completo, selezionare come Destinazione il percorso/file in cui salvare localmente il backup ed impostare l'opzione Sovrascrivi supporto esistente:

  3. Nella pagina Opzioni impostare il flag Verifica backup al termine:

  4. A questo dalla pagina opzioni si imposta il flag Pianificazione si preme il pulsante con i tre punti (...):

  5. Sulla finestra Modifica pianificazione che appare si imposta il Nome del processo, si seleziona Tipo pianificazione al valore Periodica e si preme sul pulsante Cambia:

  6. Indicare la Frequenza voluta (si consiglia una schedulazione tale che i backup vengano fatti in tutti i giorni lavorativi), l'orario (Frequenza giornaliera) e l'opzione Nessuna data di fine. Quindi si preme OK sulla finestra Modifica pianificazione processo periodico, sulla sulla finestra Modifica pianificazione e sulla finestra di impostazione del backup. Questa azione non ha fatto altro creare un nuovo processo all'interno dell'SQL Server Agent.

  7. Ritornare sulla finestra Backup di SQL Server per impostare ora il backup relativo al log delle transazioni: nella pagina Generale impostare il tipo di backup su Log delle transazioni, selezionare come Destinazione il percorso/file in cui salvare localmente il backup, file che deve essere diverso da quello in cui viene conservato il backup del database completo, ed impostare l'opzione Sovrascrivi supporto esistente. Qualora non sia disponibile il tipo di backup Log delle transazioni significa che sul database è impostato un Modello di recupero impostato a Semplice; modificare tale impostazione su Completo (dalle Proprietà del database, scheda Opzioni). Il campo Nome contraddistingue in modo univoco il backup di un database da un altro o dal suo log, per cui tali nomi devono essere sempre differenti.

  8. Nella pagina Opzioni impostare il flag Verifica backup al termine:

  9. A questo dalla pagina opzioni si imposta il flag Pianificazione si preme il pulsante con i tre punti (...):

  10. Sulla finestra Modifica pianificazione che appare si imposta il Nome del processo, si seleziona Tipo pianificazione al valore Periodica e si preme sul pulsante Cambia:

  11. Indicare la Frequenza voluta (si consiglia una schedulazione tale che i backup vengano fatti in tutti i giorni lavorativi), l'orario (Frequenza giornaliera) e l'opzione Nessuna data di fine. Quindi si preme OK sulla finestra Modifica pianificazione processo periodico, sulla sulla finestra Modifica pianificazione e sulla finestra di impostazione del backup. Questa azione non ha fatto altro creare un nuovo processo all'interno dell'SQL Server Agent.

  13. Tutti i passaggi sopra menzionati devono essere eseguiti per tutti i database di Business.
  14. Ogni processo che è stato schedulato va "provato" eseguendolo in modo manuale scegliendo la voce di menu Avvia processo

  15. Occorre a questo punto procedere con una schedulazione (con qualsiasi programma di backup) che provveda alla copia dei file di backup sopra realizzati su un dispositivo di backup esterno (tape, disco esterno, ...)
