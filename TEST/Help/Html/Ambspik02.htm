Installazione della suite Business Mobility

Installazione/configurazione lato server

  • Installare Business con database SQL Server oppure MSDE.
  • Condividere sul server la cartella \ASC di Business in modo che sia raggiungibile, tramite la rete, dai terminalini.
  • Per i moduli che necessitano di BusServer (Picking su palmare, ricevimento merci su palmare): inserire in esecuzione automatica sul server il programma BusServ.exe in modo che sia sempre attivo. Questo passaggio è indispensabile per il funzionamento delle stampe da palmare e per la creazione del documento DDT ricevuto o fattura ricevuta. Nel caso di firewall impostati sul server si rammenta che dovrà essere lasciata aperta la porta di comunicazione socket 1001.
  • Definire per gli articoli da gestire almeno un codice a barre.
  • Per il solo modulo di Picking su palmare impostare nel 'Registro di Business' le seguenti proprietà:
    • NotaPrelCreaPicking con valore -1 se si vuole che all'emissione di una nota di prelievo venga generata automaticamente la picking list.
    • GestioneMatricSuNoteDiPrel con valore 2, inquanto nel caso di articoli gestiti a matricole, è obbligatorio inserire la matricola nella nota di prelievo.

Requisiti del Pocket PC

  • Sistemi operativi supportati: Windows Mobile 2003, Windows Mobile 2005, Windows CE 4.2, Windows CE 5.0. Va specificato che i sistemi Windows CE sono supportati ma con alcune limitazioni, per es. l'emissione dei suoni a conferma del barcode letto.
  • Wi-Fi o altro strumento che permetta dal palmare di sfogliare la rete per raggiungere la cartella condivisa sul server.
  • Lettore codici a barre per la lettura dei barcode degli articoli.
  • Deve essere installato il Media Player per la riproduzione dei suoni (.wave).

Installazione lato palmare

Per l'installazione del modulo sul Pocket PC è necessario utilizzare il CD di installazione di 'BusMobi', unico per tutti i programmi della suite Binuess Mobility.

All'interno del CD sono presenti dei file con estensione .CAB per l'installazione del programma e un file leggimi.txt con alcune note per l'installazione.

Copiare i file con estenzione .CAB sul palmare in una cartella temporanea (per esempio 'My Documents') e procedere con l'installazione selezionandoli uno alla volta nel seguente ordine:

1 - Framework di Dot.net 2.0, a seconda del sistema operativo installato sul palmare bisognerà selezionare:

  • per Windows Mobile 2003: NETCFv2.ppc.armv4.cab
  • per Windows Mobile 2005: NETCFv2.wm.armv4i.cab
  • per Windows CE 4.2: NETCFv2.wce.armv.cab
  • per Windows CE 5.0: NETCFv2.wce5.armv4i.cab

2 - Supporto lingua italiana per il framework di Dot.net 2.0: System_SR_IT.CAB

3 - File per SQL CE:

  • sql.dev.ENU.ppc.wce4.armv4.CAB
  • sql.ppc.wce4.armv4.CAB

4 - File per installazione del programma: busmobi_setup.CAB

Al termine dell'installazione del programma, bisogna configurarlo per poter lavorare con Business. Per fare ciò modificare le proprietà contenute nel file buspick.ini (vedi sotto) presente su palmare nella directory /dispositivo/programmi/buspick/ (directory creata dal programma di installazione). Si consiglia, una volta impostato, di tenere sul server (o su altro pc), una copia di questo file 'buspick.ini' per velocizzare il ripristino del terminale in caso di reset.

Inoltre è necessario rinominare il palmare dandogli un nome non più lungo di 6 caratteri (es: term1, term2, ...). Per fare ciò agire da avvio->impostazioni->sistema->informazioni su->ID dispositivo->nome dispositivo

Verificare nelle impostazioni internazionali che la lingua sia 'Italiano'.

Infine, è necessario configurare lo scanner dei barcode in modo che:

  • la proprietà 'Postamble' sia '\R', ovvero dopo la lettura del barcode deve dare un ENTER
  • la proprietà 'Global symbology ID' sia 'disabled' (questo è necessario affinchè dopo la lettura del barcode, il palmare non aggiunga nessun carattere per identificare la tipologia)

L'impostazione di queste proprietà varia a seconda del palmare ed in particolare dello scanner di barcode presente.
