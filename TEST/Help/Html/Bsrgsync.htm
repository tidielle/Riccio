Sincronizzazione con il server centrale

Nome programma: BNRGSYNC.DLL
Moduli richiesti: Gestione punti vendita recovery
Opzioni programma: vai alle opzioni
Report programma: vai ai report

Questo programma, per le installazioni di tipo negozio è richiamabile solo dal Menu gestione punti vendita, provvede prima a generare un file contenente i dati del negozio corrente (movimenti) da inviare alla sede centrale, prefissonegozio_aaaammgg_hhmm.send, e poi ad importare i dati aggiornati inviati dalla sede centrale (clienti, articoli, condizioni commerciali, ecc...), prefissonegozio_aaaammgg_hhmm.neg.

Modifica
Permette di modificare le impostazioni di sincronizzazione del negozio
NB: le modifiche devono essere riportate anche lato server per evitare la sovrascrittura alla prima sincronizzazione server ->negozio

Tipo sincronizzazione

Completa
Permette di inviare i dati del negozio corrente e ricevere i dati aggiornati dalla sede centrale

Solo Invio
Permette di inviare i dati aggiornati del negozio corrente, in fase di invio verrà creato un file prefissonegozio.lock che blocca ogni altra attività di sincronizzazione sia dal server (solo per il negozio corrente), che per il negozio corrente; il programma controlla se è presente nella cartella condivisa il file prefissonegozio_*.ok (nome e data dell'ultimo file sincronizzato correttamente dalla sede centrale) e se presente provvede a spostare nella cartella OLD_SYNC del server tutti i file precedentemente sincronizzati, infine provvederà a creare un nuovo file prefissonegozio_aaaammgg_hhmm.send ed eliminati i file .lock generati precedentemente.

Solo Ricezione
Permette di ricevere i dati aggiornati della sede centrale, in fase di ricezione verrà creato un file prefissonegozio.lock che blocca ogni altra attività di sincronizzazione sia dal server (solo per il negozio corrente), che per il negozio corrente; il programma controlla se è presente nella cartella condivisa il file prefissonegozio_*.neg (file generati dalla sede centrale per il negozio) e se presente provvede ad importare i dati dal file con data più recente. A fine elaborazione verranno spostati tutti i file .neg nella cartella OLD_SYNC del server e creato il file prefissonegozio_aaaammgg_hhmm.recok.

NB: dopo ogni sincronizzazione verranno creati i file di log: BNRGSYNC_Invio_operatore.log e BNRGSYNC_Ricezione_operatore.log
