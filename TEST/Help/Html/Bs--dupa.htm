Gestione Aziende/Database

Nome programma: BS--DUPA.DLL
Moduli richiesti: Qualsiasi

Questo programma specifica il tipo di archivio utilizzato per conservare i dati dell'azienda corrente, permette di creare una nuova azienda, modificare ed effettuare altre operazioni su un'azienda già esistente.

Codice azienda Indicare il codice dell'azienda da creare oppure su cui ricopiare tabelle o archivi anagrafici da altra azienda. Il codice d'attribuire può essere composto esclusivamente da lettere e numeri, massimo 12 caratteri senza spazi. Nomi validi per un'azienda sono i seguenti: "1", "ADF4", "4XCL", "ALFA". Nomi non validi sono ad esempio: "$FFF", "A HH1", "R.UU", "ALFà".

Codice ditta predefinito Indica il nome dell'unica ditta presente o da creare nel database monoditta, oppure, con database multiditta, nome e codice della ditta predefinita. Se il database è monoditta, il codice della ditta predefinita dev'essere uguale al nome del database.

Non proporre questo databse selezionato negli aggiornamenti se spuntato le operazioni di conversioni che partono in automatico durante gli aggiornamenti non vengono effettuate per questo database, sarà l'utente a lanciarle manualmente.

Ragione sociale Questo campo contiene la ragione sociale dell'azienda corrente, in fase di nuovo inserimento digitare l'esatta ragione sociale dell'azienda da creare.

URL sito azienda È utilizzato da Business 4 People per avvisare tramite e-mail di nuovi post, inserendo il link diretto a Web Business Net per consultare il post in modo pratico. Ad esempio, se si memorizza http://www.ntsinformatica.it, nell'email del post sarà aggiunto "/wbn.aspx?DB=...".

Stringa Connessione se il database è di tipo SQL Server indicare una stringa del tipo:

Server=nome_server;Database=nome_azienda;UID=sa;pwd=password;APP=Business

dove la stringa nome_server va sostituita col nome della macchina che fa da server, la stringa nome_azienda va sostituita col codice dell'azienda e la stringa password con la password per l'utente sa. Si può indicare nella stringa di connessione il connection timeout in secondi (se non specificato è di 30), ad esempio:

Server=PC-SERVER;Database=PROVA;UID=sa;pwd=ab-cd;LANGUAGE=us_english;APP=Business;Connect Timeout=50;

Creazione di un nuovo database

Per creare un nuovo database è possibile utilizzare la procedura guidata, il programma propone una serie di maschere dove inserire i dati per comporre in modo automatico la maschera. La prima maschera richiede i dati relativa all'azienda da creare.

La seconda maschera richiede le informazioni per collegarsi all'istanza di SQL (server/istanza, autenticazione, ecc.).

Nell'ultima maschera viene chiesto che operazione eseguire:

Se viene selezionato Crea nuovo database viene lanciata l'operazione di creazione struttura che consente di creare un database vuoto nella fase di inserimento di una nuova azienda. La procedura provvedere a creare il database SQL, infine viene chiesto se cancellare o meno le tabelle standard distribuite da NTS. Se si risponde affermativamente nel nuovo database non saranno presenti codici IVA, causali contabili e piani dei conti standard.
Successivamente viene chiesto se, al momento della creazione di nuovo sottoconto PDC deve venir creata una contropartita contabile con descrizione uguale a quella del nuovo conto contabile appena creato. Le contropartite PDC vengono utilizzate, principalmente, nel momento in cui nel database vengono inserite più di una ditta. Nel dubbio rispondere affermativamente alla questa domanda. NB: il flag 'Caratteri Unicode' serve a creare un database con campi NVARCHAR invece che VARCHAR, per poter scrivere descizioni per lingue straniere come ideogrammi cinesi, russo, ...

Se viene selezionato Crea da database esistente non viene lanciata alcuna operazione in quanto probabilmente il database è già presente su SQL Server (magari si è ripristinato un backup).

Se viene selezionato Collega il file .MDF/.LDF a Server SQL consente di agganciare un database/azienda già esistente nella fase di inserimento di una nuova azienda. Per collegare l'azienda occorre copiare i file nome_azienda.MDF e nome_azienda.LDF del database dell'azienda da aggiungere nella directory Data di installazione di SQL Server/MSDE (per esempio C:\PROGRAM FILES\MICROSOFT SQL SERVER\MSSQL10_50.CORSO\MSSQL\DATA\).

Gestione backup

Questo pulsante consente di accedere alla finestra di Gestione backup che permette di gestire le procedure di backup di tutti i database/azienda e del database/procedura (solitamente ARCPROC).

Importa dati da altro DB

Questo pulsante consente di accedere alla finestra di Importa dati da altro DB che permette di importare dati da un database/azienda ad un altro.

Crea server DSN

Questo pulsante consente di creare automaticamente una Fonte dati ODBC (DSN=Data Source Name) nel pannello di controllo ODBC 32. Questa operazione è necessaria solo se tale fonte dati non è stata creata in altra occasione (per esempio inserendo una nuova azienda) oppure se è stata cancellata. Un DSN è necessario per operare con le stampe di Business. Il DSN viene creato di sistema se l'installazione di Business è per macchina, altrimenti viene creato per utente. Con il DSN di sistema è possibile eseguire stampe da Web Business Net, inoltre se sul pc si configurano nuovi utenti di windows (o utenti terminal server) non è più necessario loggarsi con questi ultimi e creare il server DSN da dentro Business per poter stampare senza errori.

Ricrea stored procedures

Questo pulsante permette di ricreare le stored procedures, senza dover eseguire le procedure di conversione archivi.
