Gestione dei backup da operazioni pianificate di Windows

La schedulazione dei backup attraverso le Operazioni pianificate di Windows è necessaria solo con SQL Server 2005/2008 Express Edition dove non è disponibile l'SQL Server Agent.

L'utilizzo delle operazioni Pianificate rende comunque più critico e vulnerabile il processo backup per il fatto che talvolta tali processi si possono bloccare e non vi sono efficaci notifiche di eventuali errori occorsi. Pertanto va valutata bene l'adozione di SQL Server 2005/2008 Express Edition: in installazione critiche è opportuno utilizzare SQL Server 2005/2008 Standard Edition.

L'impostazione della Operazione pianificata deve avvenire sul server SQL, ed entrando nelle Operazioni pianificate (da Pannello di Controllo) assicurarsi che in Avanzate vi sia la spunta su Notifica operazioni non eseguite e che l'Utilità di pianificazione non sia disattivata. Per verificare l'esito di un processo schedulato si sceglie la voce di menu Avanzate, Visualizza registro.
