Generazione Ratei e Risconti

Nome programma: BSCGGRAT.DLL
Moduli richiesti: Contab. generale e sempl.
Opzioni programma: vai alle opzioni
Report programma: vai ai report

Questo programma genera automaticamente i Ratei ed i Risconti sia in Prima nota ufficiale che sotto forma di registrazioni di Prima nota Integrativa. Dal menù Strumenti è possibile scegliere le sorgenti dei dati per l'elaborazione: gli archivi di contabilità o gli archivi di contabilità integrativa.

L'elaborazione può essere eseguita più volte, sia in modalità ordinaria che integrativa, perciò:

- sono considerate le registrazioni relative a sottoconti che hanno il campo "Richiedi date" impostato a Data competenza economica, per costi e ricavi anticipati e costi e ricavi posticipati, oppure Periodo competenza economica, per ratei e risconti attivi e passivi.

- non sono considerate le registrazioni relative a ratei/risconti già presenti in archivio, riconosciute dal Tipo movimento indicato nelle causali utilizzate.

In fase d'inserimento documenti da prima nota, occorre preventivamente gestire le date di competenza economica sulla griglia del castelletto contropartite sulle colonne "Da Data Comp. Ec." e "A Data Comp. Ec.".

Il programma gestisce tutte le otto tipologie - Risconti, Ratei, Costi/Ricavi anticipati, Costi/Ricavi posticipati, divise poi per attive e passive - eseguendo 4 elaborazioni differenti, una per ciascuna delle seguenti combinazioni:

1 Risconti e costi/ricavi anticipati, attivi Costi rilevati in via anticipata Considera le registrazioni con data registrazione fino alla data indicata nel campo "Calcola alla data"
2 Risconti e costi/ricavi anticipati, passivi Ricavi rilevati in via anticipata Considera le registrazioni con data registrazione fino alla data indicata nel campo "Calcola alla data"
3 Ratei e costi/ricavi posticipati, attivi Ricavi rilevati in via posticipata Analizza le registrazioni effettuate con data posteriore alla data indicata nel campo "Calcola alla data"
4 Ratei e costi/ricavi posticipati, passivi Costi rilevati in via posticipata Analizza le registrazioni effettuate con data posteriore alla data indicata nel campo "Calcola alla data"

Si noti come nei primi due casi considera tutte le registrazioni con data registrazione fino alla data indicata nel campo "Calcola alla data", mentre negli altri due casi analizza tutte le registrazioni effettuate con data posteriore a quella indicata nel campo "Calcola alla data".

Per l'elaborazione, il programma utilizza i conti e le causali contenuti nella tabella Personalizzazione contabilità generale, che perciò deve contenere tutti i riferimenti necessari. Nel caso di indicazione di codici di contropartita, detti codici vanno correlati direttamente ai conti del PDC utilizzato dalla ditta. Le causali indicate devono essere caratterizzate da Tipo movimento valorizzato su Ratei/Risconti.

Si genera una registrazione di risconti solo se, nell'ambito delle registrazioni del periodo selezionato, per una determinata registrazione:

  • la data della registrazione è anteriore alla data fine competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo anticipato, e la data reg. indicata nella maschera è compresa tra la data inizio competenza economica e la data fine competenza economica indicate sulla riga della registrazione contabile;
  • la data della registrazione precede o è uguale alla data di elaborazione.

Si genera una registrazione di costi-ricavi anticipati solo se, nell'ambito delle registrazioni del periodo selezionato, per una determinata registrazione:

  • la data della registrazione è anteriore alla data di competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo anticipato, e la data reg. indicata nella maschera è anteriore alla data di competenza economica;
  • la data reg. indicata nella maschera è posteriore o uguale alla data della registrazione originaria.

Una registrazione di risconti è generata solo se:

  • la data della registrazione è anteriore alla data fine competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo anticipato, e la data reg. indicata nella maschera è compresa tra la data inizio competenza economica e la data fine competenza economica indicate sulla riga della registrazione contabile;
  • la data della registrazione precede o è pari alla data di elaborazione.

Una registrazione di costi-ricavi anticipati è generata solo se:

  • la data della registrazione è anteriore alla data di competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo anticipato, e la data reg. indicata nella maschera è anteriore alla data di competenza economica;
  • la data reg. indicata nella maschera è posteriore o pari alla data della registrazione originaria.

Una registrazione di ratei è generata solo se:

  • la data della registrazione è posteriore alla data di fine competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo posticipato, e la data registrazione indicata nella maschera è uguale o anteriore alla data di fine competenza economica;
  • la data della registrazione è uguale alla data di fine competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo posticipato, e la data registrazione indicata nella maschera è anteriore alla data di fine competenza economica;
  • la data della registrazione è posteriore alla data di fine competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo posticipato, e la data registrazione indicata nella maschera è anteriore alla data di fine competenza economica;
  • il costo è registrato con data posteriore alla data fine competenza economica e la data di elaborazione è posteriore alla data fine competenza economica.

Sono escluse le registrazioni nelle quali la data inizio competenza economica é superiore alla data elaborazione.

In particolare il programma genera un'operazione di rateo nel caso in cui il campo "Calcolo alla data" rientri nella data di competenza economica dell'operazione in prima nota. In questo caso il programma di Generazione Ratei considera l'eventuale competenza di periodi precedenti, come fosse corrente. Sarà quindi l'operatore ad effettuare, manualmente, un'operazione di rettifica in contabilità generale. Questa particolarità nasce dall'esigenza di lasciare inalterato il risultato economico dell'esercizio corrente, poichè si ritiene opportuno mantenere il costo di competenza dell'esercizio precedente che gravi su quello corrente. L'operatore poi valuta un eventuale giroconto della quota di competenza dell''anno precedente alla voce sopravvenienze passive.

Per ottenere un bilancio infrannuale con i dati economici di competenza corretti, secondo l'esercizio in corso, è consigliabile lanciare il programma "Stampa bilancio contabile" con le opzioni "Calcola ratei e risconti" ed "Esegui il calcolo anche alla data di inizio".

Se in "Anagrafica ditta - Dati aggiuntivi contabilità" sono attivi i flag "Gestione stanziamenti" e "Ripartisci costi/ricavi su mese", la generazione dei ratei e risconti non elabora i ratei e calcola diversamente gli importi oppure non elabora i ratei e risconti se non sono stati ricalcolati i costi mensili.

Esempio

Periodo Esercizio Corrente 01.01.n – 31.12.n

Data registrazione in prima nota della operazione di costo gestito per Periodo di Competenza: 28.02.n

Periodo di competenza del costo: 01.03.n-1 – 28.02.n

Se lancio la Generazione Ratei e risconti con data registrazione fuori dal periodo di competenza, il sistema NON genera alcun rateo/risconto, quindi tutto il costo graverà sul periodo "n". L'operatore deve intervenire manualmente.

Se lancio la Generazione Ratei e Risconti con data reg. 31.01.n, il sistema GENERA un rateo per il periodo dal 01.03.n-1 al 31.01.n, effettuando un'operazione al 31.01.n e una al 01.02.n per la quota di competenza dal 01.02.n-28.02.n. Nessuna operazione è eseguita per lo storno dal 01.03.n-1 al 31.12.n-1, per non alterare il risultato economico dell'esercizio. L'operatore deve intervenire manualmente, secondo le proprie esigenze contabili, cioè se l'anno precedente è già chiuso o da chiudere.

La registrazione che va fatta manualmente in caso di doppio esercizio in fase di registrazione della fattura al 28.02.n è:

28/02/n      Diversi    a      FORNITORE
             costo
             Fatture fornitore da ricevere

31/12/n-1    Costo      a      fatture da ricevere.

In questo caso, al lancio dei ratei e risconti, il calcolo è corretto anche dal punto di vista formale.

Una registrazione di costi-ricavi posticipati è generata solo se:

  • la data della registrazione è posteriore alla data di competenza economica della riga di registrazione, ed è pertanto un costo o ricavo registrato in modo posticipato;
  • la data reg. indicata nella maschera è anteriore alla data di registrazione.

L'esecuzione di questo programma non genera alcuna registrazione in contabilità analitica: dev'essere utilizzata, a tal scopo, l'apposita utilità "Genera Ratei e Risconti di Contab. Analitica".

Selezione ditte Indicare se operare sulla ditta corrente oppure su un elenco o una selezione di ditte effettuata col pulsante "Seleziona".

Cancella eventuali registrazioni generate precedentemente [...] Se in precedenza un'elaborazione ratei/risconti è stata eseguita alla stessa data e con le stesse causali, spuntando questa casella il programma prima cancella tali registrazioni dall'archivio prima nota scelto; equivalente a rielaborazione.

Calcolo alla data Indicare la data di fine periodo a cui il calcolo deve essere riferito; ad esempio, per determinare ratei e risconti del primo semestre 2014 indicare 30/06/2014.

Con la funzionalità che separa i costi/ricavi gestiti a periodo di competenza economica direttamente al salvataggio della registrazione, suddivisi per mesi, il programma di generazione ratei/risconti è stato modificato perchè, se la data di elaborazione corrisponde ad un fine mese, il calcolo dei ratei/risconti non avvenga secondo le regole classiche di ripartizione del costo/ricavo per giorni di competenza, ma sia letto direttamente dalla tabella di ripartizione per mesi. É possibile che il calcolo non corrisponda al centesimo col sistema di calcolo classico, soprattutto nell'ultimo mese, in cui con la nuova tecnica l'importo è calcolato per differenza tra importo totale registrato e quote inserite nei mesi precedenti. Per eseguire il calcolo col metodo classico, si disabilita il flag Ripartisci competenza costi/ricavi su mese in Anagrafica ditta - Dati aggiuntivi contabili e s'esegue l'elaborazione; poi si riabilita il flag.

Data registrazione in Prima Nota Indicare con quale data di registrazione inserire i movimenti generati. Di norma si indica la stessa data indicata per il calcolo.

Scelta esercizio Indicare se le registrazioni da generare appartengono all'esercizio corrente o a quello precedente, coerentemente alla data di registrazione inserita.

Genera registrazioni di storno Quest'opzione genera anche le registrazioni di storno nell'esercizio successivo o nel periodo contabile, mese o trimestre, successivo. Vanno inseriti la data di registrazione e l'esercizio di competenza utilizzati per le registrazioni di storno generate in automatico. Ad esempio, se nel campo Data registrazione è indicato 31/03/2015, cioè la fine del primo trimestre, in questo campo è consigliabile indicare come data 01/04/2015, in modo che le registrazioni dei ratei e risconti del primo trimestre siano compensate all'inizio del trimestre successivo.
