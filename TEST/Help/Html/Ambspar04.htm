La gestione del modulo parcellazione

La gestione, una volta completata la fase di impostazione iniziale, si compone delle seguenti funzioni:

  • inserimento delle attività effettuate nello studio nei confronti/per conto dei clienti dello studio stesso (opzionale)
  • Stampa di controllo (o stampe di resoconto o estratto conto per il cliente) delle attività eseguite (filtrate per periodo, per cliente, per dipendente, ecc.)
  • emissione di notule (cioè di avvisi di parcella) e di parcelle, prelevando o meno dall'archivio dalle attività eseguite (precedentemente registrate)
  • contabilizzare (portare in contabilità) le parcelle (e le notule) (vedi più avanti, con maggior dettaglio) (anche in formato Word o PDF) ed inviarle (opzionalmente) ai destinatari tramite fax o posta elettronica
  • caricare ed aggiornare lo scadenzario degli adempimenti dello studio
  • effettuare stampe di controllo delle scadenze dallo scadenzario di studio, ed emettere (opzionalmente) lettere/documenti (in formato Word) per es. per avvertire i clienti interessati all'adempimento stesso 8anche qui potendo inviare automaticamente via posta elettronica o via fax il documento generato
  • procedere periodicamente al rinnovo di quelle scadenze già scadute, che si rinnovano periodicamente (es. nomina amministratori, versamento contributo CCIAA, ecc.)

Un discorso più approfondito si richiede per la parte relativa alla contabilizzazione delle parcelle e delle notule.

Si intende con ciò anche la gestione ed il giro del portafoglio RB , sia per le fatture che per le notule.

A tal scopo occorre prevedere (e collegare a conti del PDC della ditta o utilizzato in quella ditta) le seguenti contropartite (a fianco i codici delle contropartite standard) :

  • Effetti attivi (1044) gestito a partite e scadenze
  • Clienti C/RB (1102) ( gestito a partite e scadenze) (è un unico sottoconto generico, per tutti i clienti)
  • Clienti Notule (1100) gestito a part/scad.
  • Conto transitorio notule (1101) (non gestito a partite/scadenze, con tipo cronologico 'Cassa/banca')

Occorre inoltre indicare questi codici di contropartita nella tabella 'Personalizzazione Contab. Generale' (1.6.M del menù principale).

Se la ditta è un professionista i meccanismi di gestione dei ricavi per cassa anzichè per competenza, impone che la fattura emessa nei confronti di un cliente non risulti incassata fintanto non sia effettivamente tale (cioè a buon fine della ricevuta bancaria). Perciò il giro contabile impone in questi casi la non chiusura del cliente (e della scadenza associata al cliente) con l'emissione della ricevuta bancaria. Per far si che Business operi (e solo per i professionisti con il modulo Parcellazione installato) in questo modo, occorre in anagrafica ditta , dati aggiuntivi contabili, settare opportunamente il campo 'Emissione effetti con chiusura cliente' , togliendo la spunta.

Le causali utilizzate nella contabilizzazione delle fatture e notule (nonchè note di accredito ) devono essere indicate nella tabella 'Personalizzazione Parcellazione', in particolare le causali standard a tal scopo sono le seguenti :

Se ditta professionista (caso più comune) :
Emissione parcella non incassata : 1010
Emissione nota accr. Storno fattura : 1031
Emissione nota accredito : 1030
Emissione corrispettivi : 1110
Incasso fattura : 2000
Emissione notula : 1160

Se ditta non professionista (es. agente di commercio) :

Emissione parcella non incassata : 1000
Emissione nota accr. Storno fattura : 1020
Emissione nota accredito : 1020
Emissione corrispettivi : 1100
Incasso fattura : 2000
Emissione notula : 0 (non è possibile la gestione di notule per non professionisti)

inoltre (per entrambi i tipi di ditte) nella tabella personalizz. Contabilità generale saranno previste :

Emissione effetti : 3102
Presentazione distinte : 3103

Si utilizzeranno inoltre, nel giro contabile , anche le seguenti causali :

Accredito in banca di distinta : 3105 (GIROCONTO BANCA SBF-CC ) (sia per anticipo sbf che al dopo incasso)
Insoluto : 3112 (nella gestione senza chiusura cliente)
Incasso fattura alla scadenza : 3113 (nella gestione senza chiusura cliente)

Il giro contabile , nel caso professionista, nei vari casi è il seguente

CASO A : fattura con pagamento non ricevuta bancaria (es. rimessa diretta)

Operazione : Emissione fattura
Programma utilizzato : contabilizzazione parcelle
Causale utilizzata : 1010 FATT CLI NON INC. (Prof)
Dare : Cliente (si apre in scadenziario una o più scadenze sul cliente)
Avere : ricavi sospesi e Iva su vendite

Operazione: Incasso della fattura
programma utilizzato : gestione P.Nota
Causale utilizzata : 2000 INC. CLIENTE
Dare : Cassa
Avere : Cliente (si chiude la scadenza del cliente in scadenzario) (e inserisce dati per registro cronologico/gest. Profess.)
Dare automatico : Ricavi sospesi
Avere automatico : Ricavi (es. COMPENSI: ASSOGG. A RITENUTA)

CASO B : fattura con pagamento ricevuta bancaria

Operazione : Emissione fattura
programma utilizzato : Contabilizzazione Parcelle
Causale utilizzata : 1010 FATT CLI NON INC. (Prof)
Dare : Cliente (si apre in scadenziario una o più scadenze sul cliente; nel campo 'controp.' Della scadenza viene riportato il cod. Conto Del Cliente )
Avere : ricavi sospesi e Iva su vendite

Operazione : Emissione RB
programma utilizzato : Generazione effetti
Causale utilizzata : 3102 EMESSI EFFETTI
Dare : Effetti attivi (si apre in scadenziario una o più scadenze su questo conto)
Avere : Clienti C/RB

Operazione : Presentazione distinta RB
programma utilizzato : Gestione distinte effetti/incassi/pagamenti
Causale utilizzata : 3103 PRESENT. DISTINTA EFFETTI
Dare : Banca C/sbf
>Avere : Effetti attivi (si chiude in scadenziario la scadenza presente su questo conto)

Operazione: Accredito SBF da parte della banca (se anticipato rispetto alla scadenza)
programma utilizzato : Gestione prima nota
Causale utilizzata : 3105 GIROCONTO BANCA SBF-CC
Dare : Banca C/C
Avere : banca C/SBF

Operazione : Insoluto
programma utilizzato : Gestione p.nota (non può essere utilizzato il programma 'Gest. Insoluti'!)
Causale : 3112 INSOLUTO EFFETTI CLI(NO CH.CL)
dare : Clienti c/rb
Avere : banca c/sbf (o banca c/c, se in precedenza anticipato l'SBF)
Dare : Oneri bancari (per le spese)
Avere : banca c/c

Operazione : incasso fattura (a buon fine certo della ricevuta bancaria presentata)
programma utilizzato : gestione prima nota
Causale : 3113 INCASSO RIBA CLIENTI(NO CH.CL)
Dare : Clienti c/rb
Avere : Cliente (si chiude la scadenza sul conto cliente)
Dare : ricavo sospeso
Avere : ricavo (COMPENSI ASSOG. A RITENUTA...)

CASO C : notula + fattura con pagamento non ricevuta bancaria (es: rimessa diretta)

All'atto dell'emissione della notula non si provvede a registrare niente in prima nota, nè in scadenzario (in questo caso non è necessario registrare dati in scadenzario in quanto non è necessrio procedere con il giro degli effetti). Dopo aver incassato (rimessa diretta) l'importo relativo alla notula, si provvede ad emettere la fattura e si provvede a contabilizzare la fattura e registrare il pagamento (da gestione p.nota) come nel caso A. NB: una notula non può essere incassata in modo parziale. L'incasso deve essere in unica soluzione.

CASO D : notula + fattura con pagamento ricevuta bancaria

Operazione : emissione Notula
Programma utilizzato : Contabilizzazione parcelle
Causale : 1160 EMESSA NOTULA
Dare : Clienti notule (si apre una scadenza su questo conto, nel campo 'controp.' Della scadenza viene riportato il cod. Conto Del Cliente, e la scadenza è contrassegnata come 'RB da notula')
Avere : Conto transit. Notule
NB: la notula può avere solamente un pagamento con una sola rata !!!

Operazione : Emessa RB su notula
Causale : 3102 EMESSI EFFETTI CLIENTI
Dare : Effetti attivi (si apre una scadenza su questo conto)
Avere : Clienti notule (si chiude la scadenza su questo conto)

Operazione : presentazione distinta
programma utilizzato : Gestione distinte effetti/incassi/pagamenti
Causale utilizzata : 3103 PRESENT. DISTINTA EFFETTI
Dare : Banca C/sbf
Avere : Effetti attivi (si chiude in scadenziario la scadenza presente su questo conto)

Operazione: Accredito SBF da parte della banca (se anticipato rispetto alla scadenza)
programma utilizzato : Gestione prima nota
Causale utilizzata : 3105 GIROCONTO BANCA SBF-CC
Dare : Banca C/C
Avere : banca C/SBF

Operazione : Insoluto
programma utilizzato : Gestione p.nota (non può essere utilizzato il programma 'Gest. Insoluti'!)
Causale : 3112 INSOLUTO EFFETTI CLI(NO CH.CL)
Dare : Conto transitorio notule
Avere : banca c/sbf (oppure banca c/c, se anticipato in precedenza il SBF)
Dare : Oneri bancari (per le spese)
Avere : banca c/c

Quando si è avuta certezza del buon fine della ricevuta bancaria, invece , si provvede ad emettere la fattura (utilizzando il programma 'Gestine Parcelle/notule') e quindi :

Operazione : Emissione fattura
programma utilizzato : Contabilizzazione Parcelle
Causale utilizzata : 1010 FATT CLI NON INC. (Prof)
Dare : Cliente (si apre in scadenziario una o più scadenze sul cliente)
Avere : ricavi sospesi e Iva su vendite

Operazione : Incasso fattura
programma utilizzato . Gestione P.Nota
Causale utilizzata : 3113 INCASSO RIBA CLIENTI(NO CH.CL)
Dare : conto transit. Notule
Avere: Cliente (si chiude la scadenza sul cliente) (attenzione : nella gestione professionisti/cronologico indicare come conto cassa/banca il conto della banca !)
Dare automatico : ricavi sospesi
Avere automatico : Ricavo (COMPENSI ASSOGG. ..)

e poi, con altra registrazione (sempre in seguito all'incasso), per completare contabilimente il giro, qualora gli effetti non siano stati anticipati dal SBF in precedenza (o per la parte non anticipata in precedenza):

programma utilizzato : Gestione P.Nota
Causale : 3105 GIROCONTO BANCA SBF-CC (di fatto una causale qualsiasi , puramente contabile)
Dare : Cassa/banca
Avere : banca c/sbf

Il giro contabile, nel caso di impresa (es. agente di commercio) è lo stesso giro previsto per la gestione contabile di una ditta impresa con contabilità ordinaria, e cioè con la chiusura del cliente al momento della generazione della ricevuta bancaria (non si pongono, infatti, problemi di gestione per cassa e di ricavi sospesi!)

La ritenuta di acconto (ed il conto dell'attivo RITENUTE DI ACCONTO SUBITE), nelle registrazioni sopra indicate, non è stata esplicitamente evidenziata . è tuttavia da considerare sempre che la rilevazione della ritenuta subita è (e deve essere) effettuata sempre al momento dell'emissione della fattura , e non al momento dell'incasso del documento ! Pertanto anche l'emissione delle RB sarà sempre al netto della ritenuta di acconto (non è gestibile il caso di RB per importo al lordo con versamento da parte dello studio, per conto del cliente).

NB : nella gestione delle varie tabelle ed archivi (es. voci di parcellazione, personalizzazione parcellazione, gestione attività) sono stati previsti alcuni campi (es. cod. Ditta destinazione, cod. Contropartita acquisto, ecc.) per poter attivare eventualmente in futuro anche una contabilizzazione automatica delle parcelle emesse, nella contabilità dei clienti dello studio come fatture ricevute con ritenuta di acconto. Nella attuale versione di Business queste funzione non è disponibile (potrebbe esserlo in future versioni).

Un altro discorso particolare è quello relativo alla gestione delle spese anticipate per conto del cliente, da parte di un professionista .

Occorre a tal scopo distinguere innanzitutto :

  • i rimborsi spese relativi a costi generici dello studio/professionista, o a spese non documentabili/non documentate : sono di fatto ricavi per lo studio , e sono trattati in modo analogo ai conti classici come COMPENSI ASSOGG. A R.A.
  • i rimborsi veri e propri, di imposte, tasse, ecc. Sostenute dallo studio per conto del cliente, documentate/documentabili : girano contabilmente come crediti/debiti nei confronti del cliente dello studio (nel piano dei conti Standard-PR sono contenuti nel sottoconto 20.11.0011 MOVIMENTI C/TERZI SPESE ANT. C/CLIENTI , sottoconto unico, valido per tutti i clienti/terzi)

Nell'ultimo ambito (quello propriamente detto delle 'spese anticipate'), possiamo identificare due casi :

CASO A : il professionista sostiene spese per conto del cliente e successivamente addebita in parcella tali spese

In questa ipotesi il giro contabile/operativo sarà il seguente :

Operazione : sostenimento della spesa per conto del cliente
programma utilizzato : Gestione p.Nota
Causale utilizzata : 2510 PAGAM. PER ANTIC. SPESE (Prof)
Dare : Movimenti c/terzi spese antic. C/clienti (20.11.0011) (genera dati per la gestione professionisti/cronol.)
Avere : Cassa/banca

NB: l'utente può tenere memoria di queste operazioni anche senza dover registrare subito in P.nota questi dati, inserendo nell'archivio delle attività una riga con una apposita voce di parcellazione di tipo ' Spese anticipate per conto del cliente'

A questo punto si provvede periodicamente all'emissione della parcella , che conterrà nel corpo una o più righe di spese anticipate per conto del cliente (eventualmente riprese dall'elenco delle attività registrate come al punto sopra), ed alla contabilizzazione solita di detta parcella (che chiude il conto utilizzato 20.11.0011).

CASO B : il cliente anticipa il denaro al professionista, il quale successivamente paga imposte/tasse/altro per suo conto utilizzando tali importi

In questa ipotesi il giro contabile è il seguente:

Operazione : incasso importo dal cliente per la spesa da sostenere
programma utilizzato : Gestione p.Nota
Causale utilizzata : 2010 INCASSO PER ANTIC. SPESE (Prof)
Avere : Movimenti c/terzi spese antic. C/clienti (20.11.0011) (genera dati per la gestione professionisti/cronol.)
Dare : Cassa/banca

NB: anche questa operazione potrebbe essere annotata in via preliminare (prima di andare in Gestione P.Nota) nell'archivio delle attività inserendo (col relativo importo) una riga con una apposita voce di parcellazione di tipo 'Spese anticipate dal cliente'. (riga che tuttavia non sarà mai oggetto di emissione parcella!)

Operazione : sostenimento della spesa per conto del cliente
programma utilizzato : Gestione p.Nota
Causale utilizzata : 2510 PAGAM. PER ANTIC. SPESE (Prof)
Dare : Movimenti c/terzi spese antic. C/clienti (20.11.0011) (genera dati per la gestione professionisti/cronol.)
Avere : Cassa/banca

NB: come è evidente, in questo caso l'importo delle spese anticipate non viene esposto in nessuna parcella.
