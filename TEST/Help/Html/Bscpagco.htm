Data ultimo aggiornamento: 20/10/2015

Agg. configuraz. su D.B. Neu.

Questo programma permette di aggiornare i prezzi di un range di configurazioni da indicare nei filtri Da/A configuratore (se spuntato il flag "Prezzi di vendita"), tale prezzo verrà rilevato in base alla data impostata nel campo "Data di valid. per listini".

Inoltre mediante opzione di registro BSCPCONP\OPZIONI\Aggiorna_Distinta = -1 (default = 0, non attiva), il programma da la possibilità in fase di aggiornamento di una configurazione, di effettuare anche un aggiornamento (ri-allineamento) della distinta base "chiusa" a partire da quella neutra, con possibilità di stabilire se farlo o meno, riga per riga, in base ad un flag ("Aggiorna riga": default SI) da impostare sui materiali e sulle lavorazioni della distinta base "chiusa" stessa.

Praticamente con l'opzione si da la possibilità di far lavorare il configuratore in maniera opposta. Infatti chi lavora con questo modulo sa che le distinte basi vengono salvate articolo per articolo quindi ogni articolo ha la propria distinta base, e se si interviene sulla distinta base "modello" (quella neutra) le modifiche non si ripercuotono su tutti gli articoli già configurati, quindi è necessario allinearle manualmente, cioè vince la distinta base "chiusa".

Adesso invece attivando l'opzione BSCPCONP\OPZIONI\Aggiorna_Distinta (di default non è attiva) si ha possibilità di fare lavorare la distinta base "modello", come distinta base vincente, cioè se viene modificata la distinta base "modello" questa allinea tutte le distinte base chiuse già configurate, quindi ad esempio se si vuole aggiungere un materiale in tutte le distinta base chiuse nel passato il programma lo fa in automatico. NB: l'allineamento avviene in modifica, cancellazione ed inserimento.

Questo è possibile comunque derogarlo riga per riga, cioè il default abilitando l'opzione è che il programma allinea tutto, però si può anche decidere in qualche distinta base "chiusa", togliendo la spunta dal flag "Aggiorna riga" presente sia sui materiali che sulle lavorazioni, di poter andare a dire che in una distinta base "chiusa" quella riga di materiale o lavorazione non deve essere riallineato dalla distinta base "modello"; quindi se su una certa distinta base si vogliono fare delle deroghe è sufficiente dichiarare che non deve essere una riga aggiornabile.

In questa modalità di lavoro se ad esempio dovessi fare un cambio di un materiale, dovrei fare scadere la data fine validità di quel materiale e poi aggiungere una riga nuova di materiale con data inizio validità uguale alla data fine validità del vecchio materiale, in pratica bisogna lavorare senza usare la funzione cancella riga presente in distinta base.

Se si vuole invece modificare o aggiungere un materiale in un distinta base "chiusa", e non si vuole che nessuno poi tocchi quella riga, basta togliere la spunta da "Aggiorna riga" sui materiali (e/o sulle lavorazioni) della distinta base "chiusa".
